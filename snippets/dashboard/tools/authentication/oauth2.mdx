**OAuth 2.0 Authentication**

Industry-standard secure authentication that doesn't require sharing passwords.

<Steps>
<Step title="Initiate OAuth flow">
  - Go to Tools section in your Tars dashboard
  - Find the tool you want to connect
  - Click **"Connect"** or **"Add Integration"**
  - Select **"OAuth 2.0"** as authentication method
</Step>

<Step title="Authorize access">
  - You'll be redirected to the tool's authorization page
  - Log in with your tool account credentials
  - Review the permissions Tars is requesting
  - Click **"Authorize"** or **"Allow"** to grant access
</Step>

<Step title="Complete authorization">
  - You'll be redirected back to Tars
  - The system automatically exchanges the authorization code for access tokens
  - Connection is established securely
</Step>

<Step title="Verify connection">
  - Review the connection status in your dashboard
  - Test the integration to ensure it's working properly
  - Your tool is now connected and ready to use
</Step>
</Steps>

**Common tools using OAuth 2.0:**
- **Google Workspace**: Gmail, Calendar, Drive, Sheets
- **Microsoft 365**: Outlook, Teams, OneDrive, Excel
- **Slack**: Team communication platform
- **GitHub**: Code repository management
- **Salesforce**: CRM and business platform

**Benefits of OAuth 2.0:**
- **No Password Sharing**: Your credentials never leave the original service
- **Granular Permissions**: Control exactly what Tars can access
- **Automatic Token Refresh**: Tokens are renewed automatically
- **Revocable Access**: You can revoke access anytime from the tool's settings

<Note>
**Security**: OAuth 2.0 is the most secure authentication method. You can revoke access at any time from your tool's account settings.
</Note>
