**Bearer Token Authentication**

Direct token-based authentication for high-performance, stateless access to APIs.

<Steps>
<Step title="Obtain bearer token">
  - Log into your tool's admin panel or API section
  - Navigate to API settings or token management
  - Generate a new bearer token or access token
  - Copy the token (keep it secure - it won't be shown again)
</Step>

<Step title="Configure in Tars">
  - Go to Tools section in your Tars dashboard - Find the tool you want to
  connect - Click **"Connect"** or **"Add Integration"** - Select **"Bearer
  Token"** as authentication method
</Step>

<Step title="Enter token">
  - Paste your bearer token in the **"Token"** field - Add any additional
  required fields (if applicable) - Configure token prefix if required (usually
  "Bearer") - Click **"Test Connection"** to verify
</Step>

<Step title="Complete setup">
  - Review the connection details
  - Click **"Save"** to complete the integration
  - Your tool is now connected and ready to use
</Step>
</Steps>

**Common tools using Bearer Token:**

- **REST APIs**: Most modern REST APIs use bearer tokens
- **JWT Tokens**: JSON Web Tokens for stateless authentication
- **Custom APIs**: Proprietary APIs with token-based access
- **Microservices**: Service-to-service communication
- **GraphQL APIs**: GraphQL endpoints with token authentication

**Token types supported:**

- **JWT Tokens**: JSON Web Tokens with expiration
- **API Tokens**: Custom tokens generated by the service
- **Access Tokens**: OAuth access tokens (when used directly)
- **Custom Tokens**: Any token-based authentication system

**Benefits of Bearer Token:**

- **Stateless**: No server-side session storage required
- **High Performance**: Fast authentication without database lookups
- **Scalable**: Easy to scale across multiple servers
- **Flexible**: Works with various token formats and systems

<Note>
  **Token Security**: Bearer tokens provide full access to the associated
  account. Keep them secure and rotate them regularly.
</Note>

<Warning>
  **Expiration**: Many bearer tokens have expiration dates. Monitor token
  validity and set up automatic renewal if supported by the tool.
</Warning>
